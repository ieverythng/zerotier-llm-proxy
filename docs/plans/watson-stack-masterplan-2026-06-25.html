Watson Stack Masterplan

Make the local Watson/Hermes inference stack recoverable, measurable, and usable at the highest context size that still behaves well in Discord.

Current decision: keep 65k as the interactive default; treat 80k+ as high-context candidates pending prefill measurements.

Workstreams

Runtime correctness
Canonical launcher, context-aware reloads, Lazarus recovery, Hermes stale-import restart procedure.
Prefill measurement
Separate time-to-first-token from decode tok/s, sweep upward from 4k to 65k before retesting 80k+.
High-context profiles
Promote only contexts that clear usability, quality, and recovery gates.
Documentation
NAO-style layout: architecture, plans, artifacts, reports, knowledge, traces.

Immediate sequence

  1. Restore 65k through Lazarus and the canonical Windows launcher.
  2. Run smoke and functional harness checks.
  3. Run the staged prefill sweep from 4k upward.
  4. Store measured output in docs/reports/.
  5. Update the benchmark matrix artifact after real measurements exist.

Promotion criteria

  • Correct live model alias and meta.n_ctx.
  • Proxy smoke passes.
  • Watson harness passes completion and tool-fidelity checks.
  • Discord time-to-first-token is usable.
  • Lazarus can recover to 65k after failure.
