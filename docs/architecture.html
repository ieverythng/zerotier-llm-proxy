ZeroTier LLM Proxy — Architecture

Serves local LLM inference over a ZeroTier overlay network via an OpenAI-compatible API, with optional GPT-5 Oracle access through webchat2api.

Use Cases

1. Local LLM Inference (Primary)

Hermes Agent connects to the Qwen3.6 model running on Windows GPU via the Hermes Gateway → llama.cpp bridge. This is the default front model for all agent operations.

curl http://172.24.16.1:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"qwen36-turbo-hermes","messages":[{"role":"user","content":"Hello"}]}'

2. ZeroTier Remote Access

Remote machines on the ZeroTier network can reach the LiteLLM proxy at 10.88.140.94:4000, getting full OpenAI-compatible access to the local model without exposing ports publicly.

curl http://10.88.140.94:4000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <llama-api-key>" \
  -d '{"model":"qwen36-turbo-hermes","messages":[{"role":"user","content":"Hello"}]}'

3. GPT-5 Oracle (Expert Consultation)

For complex reasoning tasks, the Hermes Agent can query GPT-5 models via webchat2api proxy. Used for architecture decisions, security assessments, and verification of technical claims.

bash /home/juanbeck/Watson/scripts/ask-gpt5.sh "Should we use FSDP or DeepSpeed for this training?" gpt-5-5-thinking

4. Codex Delegation (Code Work)

Subagents spawned via delegate_task use OpenAI Codex CLI with OAuth authentication — full tool calling, terminal access, and function-calling capabilities.

Limitations

webchat2api (Oracle) Limitations:
  • No function calling — ChatGPT built-in tools (Browse, Code Interpreter, DALL-E) are UI-only features that cannot be triggered programmatically via webchat2api or any headless proxy.
  • Token revocation — OpenAI periodically rotates/revokes access tokens. When this happens, you must extract a fresh __Secure-access_token from browser DevTools and update data/accounts.json.
  • Rate limits — ~50 messages per 8-hour window for ChatGPT Plus. Resets approximately every 8 hours.
  • Text-only — No image input/output, no file uploads through the proxy.
  • ToS risk — Automated usage of ChatGPT Plus session violates OpenAI's Terms of Service. Use at your own risk.
Codex Delegation (Recommended for tool use):
  • Full function calling — Codex CLI supports tool calls, terminal execution, and file operations.
  • OAuth auth — Uses proper OAuth flow, no token revocation issues.
  • Higher rate limits — Codex tier has more generous limits than Plus session proxy.

Infrastructure Limitations

  • Single model per server — Only one GGUF model can be loaded in llama.cpp at a time. Swapping requires stopping and restarting the server.
  • WSL cannot run Windows executables — llama.cpp must run on the Windows host; WSL is used for orchestration only.
  • GPU memory — The Qwen3.6 27B Q3_K_M model uses significant VRAM. Running a second model instance in parallel is not feasible.

Model Swap Procedure

To swap the loaded model (e.g., from Qwen3.6 to Qwopus for benchmarking):

# Stop current server
powershell -File C:\Users\Admin\PROJECTS\llama-cpp-server\scripts\stop_llama_server.ps1

# Start with new model
powershell -File C:\Users\Admin\PROJECTS\llama-cpp-server\scripts\start_turbo_hermes.ps1 `
  -Profile hermes-qwen36-64k `
  -ModelPath "D:\MODELS\Qwopus-VL-3.6-27B-Q3_K_M\Qwopus3.6-27B-v2-Q3_K_M.gguf"

Or use the unified startup script with a custom model:

powershell -File C:\Users\Admin\PROJECTS\zerotier-llm-proxy\scripts\windows\Start-Qwen36ZeroTierStack.ps1 `
  -Model qwopus-3.6-27b `
  -ModelPath "D:\MODELS\Qwopus-VL-3.6-27B-Q3_K_M\Qwopus3.6-27B-v2-Q3_K_M.gguf"
